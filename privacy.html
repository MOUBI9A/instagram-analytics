Privacy Policy

Pulse Instagram Analytics is software for analyzing Instagram accounts. It can run on the user's own machine, on a self-hosted server, or on a hosted instance the user controls. Data handling depends on which mode is in use.

Data collected

  • Instagram usernames that the user explicitly looks up.
  • Public profile data and recent public posts returned by Instagram for those usernames.
  • OAuth access tokens, when the user opts into the Facebook Login / Instagram Graph API flow.
  • AI API keys for whichever AI provider the user configures (Gemini, Groq, OpenAI, or Anthropic).

Where data is stored

  • Local-server mode: Lookup results are cached on the user's machine under ./data/lookups/. Nothing is sent to a third party.
  • Hosted-serverless mode (e.g. Vercel): No persistent cache. Each request is a fresh call to Instagram from the serverless function. Nothing is stored on the host between requests.
  • Browser state: OAuth tokens, AI API keys, watchlist, concept-studio history, and theme preferences live only in the user's browser localStorage.

Data sharing

No data is shared with third parties. The dashboard communicates only with: Instagram (for lookups), Meta / Facebook (for OAuth, when the user explicitly opts in), and the user's chosen AI provider (for AI-powered features, when the user explicitly configures an API key).

Deletion

To remove cached data: delete the ./data/lookups/ folder (local-server mode) or clear your browser storage. To revoke Meta access: visit facebook.com/settings → Business Integrations.

Cookies

The dashboard itself sets no cookies. The Facebook JS SDK may set cookies during the OAuth flow if the user opts into Live API mode.
